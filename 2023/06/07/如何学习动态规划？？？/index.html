一、入门

核心：状态定义、状态转移方程

启发思路：选或不选/选哪个

三步走：

  • 思考回溯怎么写
    • 入参和返回值
    • 递归到哪里
    • 递归边界和入口
  • 改成记忆化搜索
  • 1:1翻译成递推

当然如果能直接写出状态定义和转移方程就不用这么思考啦🤣

来一个例子

入门例子

198. 打家劫舍 - 力扣（LeetCode）

1. 首先看成回溯问题【递归搜索】

从第一个房子或者从最后一个房子开始思考，因为这两个受到的约束是最小的。

比如考虑最后一个房子选还是不选，

  • 如果不选，那么问题就变成 了n-1个房子的问题
  • 如果选，那么问题就变成了n-2个房子的问题

回溯三问：

  • 当前操作：枚举第i个房子选/不选
  • 子问题：从前i个房子中得到的最大金额和
  • 下一个子问题：分类讨论
    • 不选：从前i-1个房子中得到的最大金额和
    • 选：从前i-2个房子中得到的最大金额和

dfs(i)=max(dfs(i-1),dfs(i-2)+nums[i])

  • 递归的入参是索引即第几个房子
  • nums[i]表示第i个房子的金额
  • dfs(i)表示从前i个房子中得到的最大金额和

在定义DFS或者DF数组的含义时，它只能表示从一些元素中算出的结果，而不是从一个元素中算出的结果。

这里没有把得到的金额和作为递归的入参，而是当作了返回值！

回溯的时间复杂度是指数级别，这样写肯定会超时！

2. 递归搜索+保存计算结果=记忆化搜索

优化一下时间复杂度：

从上面的搜索树中可以看出：

  • 当两次不选的时候，会递归到dfs(i-2)
  • 一次选的时候，也会递归到dfs(i-2)

那么我们可以将第一次递归到dfs(i-2)的结果保存下来，这样就不会重复计算了。

从图中可以看到优化后的搜索树只有O(n)个节点，因此时间复杂度也优化到了O(n)

那么复杂度到底怎么算？？？

时间复杂度=状态个数*单个状态所需要计算时间

时间复杂度：O(n)【在这里状态个数是O(n)的，单个状态的计算时间是O(1)的。】

空间复杂度：O(n)【使用了一个额外的数组cache】

3. 1:1翻译成迭代【递推】

从代码中可以看出max，发生在递归调用之后，也就是在递归的【归】的过程当中，才发生了实际的计算。

如图，0和1归到了2，2和1归到了3，3和2归到了4，因此我们知道要从哪些点归到哪个点的。

那么我们可以去掉递归中的【递】，只保留【归】的过程，也就是直接从最下面开始往上计算。

自顶向下算：记忆化搜索；

自底向上算：递推。

递归—>递推方法：

  • dfs函数—>f数组
  • 递归—>循环
  • 递归边界—>数组初始值

dfs(i) = max(dfs(i-1),dfs(i-2)+nums[i])

f(i) = max(f(i-1),f(i-2)+nums[i]) //为避免负数下标，可以i从2开始

f(i+2) = max(f(i+1),f(i)+nums[i]) //也可以都+2

时间复杂度：O(n)【n是nums数组的长度】

空间复杂度：O(n)

二、选和不选

2.1 0-1背包

什么是0-1背包问题呢？

有n个物品，第i个物品的体积为w[i]，价值为v[i]，每个物品至多选一个，求体积和不超过capacity时的最大价值和。

回溯三问[还是从回溯开始思考]：

  • 当前操作：枚举第i个物品选或不选
    • 选：剩余容量减少w[i]
    • 不选：剩余容量不变
  • 子问题：在剩余容量为c时，从前i个物品中得到的最大价值和
  • 下一个子问题：
    • 选：在剩余容量为c-w[i]时，从前i-1个物品中得到的最大价值和
    • 不选：在剩余容量为c时，从前i-1个物品中得到的最大价值和

dfs(i,c) = max(dfs(i-1,c),dfs(i-1,c-w[i])+v[i])

目标和

题目：494. 目标和 - 力扣（LeetCode）

灵神题解：教你一步步思考动态规划！一个视频讲透！（Python/Java/C++/Go） - 目标和 - 力扣（LeetCode）

此题看上去与0-1背包关系不大。

探究：

假设其中添加正号的数字的和我们设置为p，那么添加负号的，就是所有元素的和s减去p

那么p-(s-p)就是我们的target，化简一下即2p=s+target，所有p=(s+target)/2，那么问题就变成从nums中选择一些数字，使得它们的和恰好等于(s+target)/2的方案数。

由于问题中nums数组是整数数组，那么s+target必须是一个偶数，否则/2得到的结果肯定是一个小数，而且s+t是不可能是负数的。

那么此题就可以转换成选一些数，这些数的和恰好等于一个数的方案数了，这是0-1背包的一种常见的变形，把递归公式的最大值改成+就好了。

1. 回溯

时间复杂度：O(2的n次方*(target+S))【每个数字都有选和不选,S为nums数组的和，n为nums数组的长度】

空间复杂度：O(n*(target+S))

2.递归搜索+保存记忆结果=记忆化搜索

时间复杂度：O(n*(target+S))

空间复杂度：O(n*(target+S))

3.1:1翻译成迭代【递推】

将dfs()改成f数组，将递归改成循环。

f[i][capacity] = f[i-1][capacity]+f[i-1][capacity-w[i]]

f[i+1][capacity] = f[i][capacity]+f[i][capacity-w[i]]【为避免出现负数下标，将i+1】

时间复杂度：O(n*(target+S))

空间复杂度：O(n*(target+S))

4.空间优化：两个数组（滚动数组）

根据上述代码可以看出，每次循环，当前的f[i+1]总是由上一个f[i]得到的，所有我们是不是只需要用到两个数组就可以了。

f[i + 1][c] = f[i][c]--->f[i + 1][c] = f[i][c];f[i + 1][c] = f[i][c]

f[i + 1][c] = f[i][c] + f[i][c - nums[i]]--->f[i + 1][c] = f[i][c] + f[i][c - nums[i]];f[i] = f[i+1]

简便写法：直接f[(i+1)%2]和f[i%2]

时间复杂度：O(n*(target+S))

空间复杂度：O(target+S)

5.空间优化：一个数组

当w[i]=2的时候，举例一下

f[i+1][c] = f[i][c]+f[i][c-w[i]]

简化：f[c] = f[c]+f[c-w[i]]

f[0]=1

i:1,2,3,5,6,7,9

i+1:1,2,3,5,6,7,9

从左往右循环：

前两个不变

f[3] = f[3]+f[1] = 3+1 = 4

f[4] = f[4]+f[2] = 5+2 = 7

f[5] = f[5]+f[3] = 6+4=10【变了，不是之前的3+6=9了】

从右往左循环：

f[7] = f[7] + f[5] = 6+9 = 15

f[6] = f[6] + f[4] = 7+5 = 12

f[5] = f[5]+f[3] = 6+3 = 9

没有变，因为没有用到之前的结果

把第一个维度去掉，从后往前枚举，从target开始到x

时间复杂度：O(n*(target+S))

空间复杂度：O(target+S)

2.2 完全背包

什么是完全背包问题呢？

有n种物品，第i个物品的体积为w[i]，价值为v[i]，每个物品至多选一个，求体积和不超过capacity时的最大价值和。

与0-1背包问题的区别：这里每种物品可以重复选择。

回溯三问[还是从回溯开始思考]：

  • 当前操作：枚举第i种物品选一个或不选
    • 选一个：剩余容量减少w[i]
    • 不选：剩余容量不变
  • 子问题：在剩余容量为c时，从前i种物品中得到的最大价值和
  • 下一个子问题：
    • 选一个：在剩余容量为c-w[i]时，从前i种物品中得到的最大价值和
    • 不选：在剩余容量为c时，从前i-1种物品中得到的最大价值和

和0-1背包问题的区别：

在选择一个物品时，i是不变的，表示继续选第i种物品，所以递归就不是到i-1了，而是到i

dfs(i,c) = max(dfs(i-1,c),dfs(i,c-w[i])+v[i])

零钱兑换

题目：322. 零钱兑换 - 力扣（LeetCode）

灵神题解：322. 零钱兑换 题解 - 力扣（LeetCode）

探究：

这里是从零钱中选择一些，然后恰好和为amount，每个零钱可以选择多次，问最少需要多少个数，这其实就是完全背包的一种变形。

把每个物品的价值看成为1【v[i]=1】，把求最大价值和，改成求最小价值和，可u得到如下的状态转移：

dfs(i,c) = min(dfs(i-1,c),dfs(i,c-w[i])+v[i])

就是最终的目标是有多少个硬币，dfs函数的返回值就是最终硬币的个数，找到最小的dfs，多一个硬币，多一个1，这里的capacity就是amount，最小的capacity。

1. 回溯

时间复杂度：O(2的n次方*amount)【其中n为coins的长度】

空间复杂度：O(n*amount)

2.递归搜索+保存记忆结果=记忆化搜索

时间复杂度：O(n*amount)

空间复杂度：O(n*amount)

3.1:1翻译成迭【递推】

时间复杂度：O(n*amount)

空间复杂度：O(n*amount)

4.空间优化：两个数组（滚动数组）

和0-1背包原理一样。

时间复杂度：O(n*amount)

空间复杂度：O(amount)

5.空间优化：一个数组

f[i][c] = min(f[i-1][c],f[i][c-w[i]]+v[i])

f[i+1][c] = min(f[i][c],f[i+1][c-w[i]]+v[i])

f[c] = min(f[c],f[c-w[i]]+v[i])

当w[i]=2;v[i]=1,我们举例一下

i:0,2,2,5,6,6,9

i+1:0,2,2,5,6,6,9

从左往右：

其中正着的时候就是满足要求的，因为我们在计算f[i+1]的时候，当c改变了，本来就是由f[i+1]得到的，所以值不会发生变化，当c不改变时，由f[i]得到，因为此时c不改变，那么更不会影响了。

f[3] = min(f[3],f[1]+1) = min(2,0+1) = 1;

f[4] = min(f[4],f[2]+1) = min(5,2+1) = 3;

时间复杂度：O(n*amount)

空间复杂度：O(amount)

2.3 常见变形

  • 至多装capacity，求方案数/最大价值和
  • 恰好装capacity，求方案书/最小价值和、最大价值和
  • 至少装capacity，求方案数/最小价值和

就是边界条件改变了，在递归到底层的时候，判断是不一样的。

边界条件总结

  • 求max/min的模型里：

    • 求体积恰好为j：
    • 求max, f = 【0】+【-inf】*t
    • 求min, f = 【0】+【inf】*t
    • 最终f【j】代表体积恰好为j时的价值极值。
    • 求体积至多为j时:
    • f【0】 = 【0】+【0】*t (max/min)
    • 最终f【j】代表体积至多为j时的价值极值
    • 求体积至少为j时:
    • f【0】 = 【0】+【0】*t (max/min)
    • 同时遍历体积需要修改循环下界v->0、转移需要修改为从max(0,j-v),即
      for j in range(self.vol, -1, -1):f【j】 = merge(f【j】, f【max(j - v,0)】 + w) # 01背包
      for j in range(self.vol+1):f【j】 = merge(f【j】, f【max(j - v,0)】 + w) # 完全背包
    • 最终f【j】代表体积至少为j时的价值极值

  • 求方案数的模型里（一般要取模）:

    • 求体积恰好为j：
    • 求max, f = 【1】+【0】*t
    • 最终f【j】代表体积恰好为j时的方案数。
    • 求体积至多为j时:
    • f = 【1】+【1】*t
    • 最终f【j】代表体积至多为j时的方案数。
    • 求体积至少为j时:
    • f = 【1】+【0】*t
    • 同时遍历体积需要修改循环下界v->0、转移需要修改为从max(0,j-v),即
      for j in range(self.vol, -1, -1):f【j】 += f【max(j - v,0)】 # 01背包
      for j in range(self.vol+1):f【j】 += f【max(j - v,0)】 # 完全背包
    • 最终f【j】代表体积至多少为j时的方案数

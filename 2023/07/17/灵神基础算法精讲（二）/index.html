一、入门

核心：状态定义、状态转移方程

启发思路：选或不选/选哪个

三步走：

  • 思考回溯怎么写
    • 入参和返回值
    • 递归到哪里
    • 递归边界和入口
  • 改成记忆化搜索
  • 1:1翻译成递推

当然如果能直接写出状态定义和转移方程就不用这么思考啦🤣

时间复杂度：

状态个数*单个状态时间

例如dfs(i,j)递归函数中，状态个数就是(i,j)的个数，i有n个，j有n个，那么肯定有O(n)方的状态，如果每个状态只要一次比较大小或者求和等，那么单个状态时间就是O(1)，总的就是O(n方)

空间复杂度：

看利用的数组大小等等

1. 首先看成回溯问题【递归搜索】

从第一个房子或者从最后一个房子开始思考，因为这两个受到的约束是最小的。

比如考虑最后一个房子选还是不选，

  • 如果不选，那么问题就变成 了n-1个房子的问题
  • 如果选，那么问题就变成了n-2个房子的问题

回溯三问：

  • 当前操作：枚举第i个房子选/不选
  • 子问题：从前i个房子中得到的最大金额和
  • 下一个子问题：分类讨论
    • 不选：从前i-1个房子中得到的最大金额和
    • 选：从前i-2个房子中得到的最大金额和

dfs(i)=max(dfs(i-1),dfs(i-2)+nums[i])

  • 递归的入参是索引即第几个房子
  • nums[i]表示第i个房子的金额
  • dfs(i)表示从前i个房子中得到的最大金额和

在定义DFS或者DF数组的含义时，它只能表示从一些元素中算出的结果，而不是从一个元素中算出的结果。

这里没有把得到的金额和作为递归的入参，而是当作了返回值！

回溯的时间复杂度是指数级别，这样写肯定会超时！

探究：

假设其中添加正号的数字的和我们设置为p，那么添加负号的，就是所有元素的和s减去p

那么p-(s-p)就是我们的target，化简一下即2p=s+target，所有p=(s+target)/2，那么问题就变成从nums中选择一些数字，使得它们的和恰好等于(s+target)/2的方案数。

由于问题中nums数组是整数数组，那么s+target必须是一个偶数，否则/2得到的结果肯定是一个小数，而且s+t是不可能是负数的。

2.2 完全背包

什么是完全背包问题呢？

有n种物品，第i个物品的体积为w[i]，价值为v[i]，每个物品至多选一个，求体积和不超过capacity时的最大价值和。

与0-1背包问题的区别：这里每种物品可以重复选择。

回溯三问[还是从回溯开始思考]：

  • 当前操作：枚举第i种物品选一个或不选
    • 选一个：剩余容量减少w[i]
    • 不选：剩余容量不变
  • 子问题：在剩余容量为c时，从前i种物品中得到的最大价值和
  • 下一个子问题：
    • 选一个：在剩余容量为c-w[i]时，从前i种物品中得到的最大价值和
    • 不选：在剩余容量为c时，从前i-1种物品中得到的最大价值和

和0-1背包问题的区别：

在选择一个物品时，i是不变的，表示继续选第i种物品，所以递归就不是到i-1了，而是到i

dfs(i,c) = max(dfs(i-1,c),dfs(i,c-w[i])+v[i])

2.3 常见变形

  • 至多装capacity，求方案数/最大价值和
  • 恰好装capacity，求方案书/最小价值和、最大价值和
  • 至少装capacity，求方案数/最小价值和

就是边界条件改变了，在递归到底层的时候，判断是不一样的。

边界条件总结

  • 求max/min的模型里：

    • 求体积恰好为j：
    • 求max, f = 【0】+【-inf】*t
    • 求min, f = 【0】+【inf】*t
    • 最终f【j】代表体积恰好为j时的价值极值。
    • 求体积至多为j时:
    • f【0】 = 【0】+【0】*t (max/min)
    • 最终f【j】代表体积至多为j时的价值极值
    • 求体积至少为j时:
    • f【0】 = 【0】+【0】*t (max/min)
    • 同时遍历体积需要修改循环下界v->0、转移需要修改为从max(0,j-v),即
      for j in range(self.vol, -1, -1):f【j】 = merge(f【j】, f【max(j - v,0)】 + w) # 01背包
      for j in range(self.vol+1):f【j】 = merge(f【j】, f【max(j - v,0)】 + w) # 完全背包
    • 最终f【j】代表体积至少为j时的价值极值

  • 求方案数的模型里（一般要取模）:

    • 求体积恰好为j：
    • 求max, f = 【1】+【0】*t
    • 最终f【j】代表体积恰好为j时的方案数。
    • 求体积至多为j时:
    • f = 【1】+【1】*t
    • 最终f【j】代表体积至多为j时的方案数。
    • 求体积至少为j时:
    • f = 【1】+【0】*t

🎈回溯三问：

  • 当前操作？考虑是s[i]和t[j]选或不选
  • 子问题？s的前i个字母和t的前j个字母的LCS长度
  • 下一个子问题？
    • s的前i-1个字母和t的前j-1个字母的LCS长度【都选或者都不选（子问题一样）】
    • s的前i-1个字母和t的前j个字母的LCS长度
    • s的前i个字母和t的前j-1个字母的LCS长度

🎈递归公式：

这里dfs是一个递归函数，就是LCS长度！！！

dfs(i,j) = max( dfs(i-1,j) , dfs(i,j-1) , dfs(i-1,j-1) + 1 ) ? s[i] = s[j]

dfs(i,j) = max( dfs(i-1,j) , dfs(i,j-1) , dfs(i-1,j-1) ) ? s[i] != s[j]

dfs(i,j) = max( dfs(i-1,j) , dfs(i,j-1) , dfs(i-1,j-1) + s[i] = s[j] )

这里其实当s[i] = s[j]时，dfs(i-1,j)和dfs(i,j-1)肯定是小于等于dfs(i-1,j-1) + 1的，因为dfs(i-1,j)再往下递归，最大也是dfs(i-1,j-1) + 1，所以直接不用让dfs(i,j)=dfs(i-1,j-1) + 1就行。

同理当s[i] != s[j]时，dfs(i-1,j)和dfs(i,j-1)肯定是大于等于dfs(i-1,j-1)的，因为dfs(i-1,j)往下递归最小也是dfs(i-1,j-1)。

所以简化上式得到：

dfs(i,j) = dfs(i-1,j-1) + 1 s[i] = s[j]

dfs(i,j) = max( dfs(i-1,j) , dfs(i,j-1) ) s[i] != s[j]

编辑距离

72. 编辑距离 - 力扣（LeetCode）

🎈思路

题目要求对word1进行增删改操作，然后转换为word2，求最少操作数。

因为每一个字符都要判断一下是否进行操作，不妨我们从最后一个开始。

判断选还是不选

🎈回溯

判断最后一个字符是否相等

word1是被转换的字符串，word2是目标字符串

i是word1递归的位置，j是word2递归的位置，开始都是最后一个字符

  • word1[i]==word2[j] --> dfs(i,j)=dfs(i-1,j-1)
    • 解释一下：
    • 此时两个字符串的最后一个字符相等了，那么不需要做任何操作，直接往下递归
  • word1[i]!=word2[j] --> dfs(i,j)=min(dfs(i,j-1),dfs(i-1,j),dfs(i-1,j-1))+1
    • 解释一下：
    • 此时最后一个字符不相等，那么我们肯定要对原字符串的最后一个字符做一次操作【+1】，
    • 插入：那么此时插入的肯定是word2的最后一个字符，word2往下递归一个，word1还是当前位置，因为它的长度增加了 ，还需要判断当前字符dfs(i,j-1)

    • 替换：那么此时会将word1的最后一个字符替换为word2的最后一个字符，此时当前的两个字符串的最后一个字符肯定相等了，往下递归dfs(i-1,j-1)

三、选哪个

典型题目：

题目：300. 最长递增子序列 - 力扣（LeetCode）

📌前言：

所谓子序列，就是从数组中选择一些数，且顺序和数组中的顺序是一致的，与子数组是不同的，子数组必须是连续的。

递增子序列 IS,Increasing Subsequence

严格递增子序列 LIS,Longest Increasing Subsequence

$O(n^2)$ 回溯 –> 记忆化搜索 –> 递推

$O(nlog_n)$ 二分+贪心

思路1

📌思路1：

由于子序列本质上就是数组的一个子集，那么我们就用子集型回溯来思考；

对于子集型回溯，我们有【选/不选】和【枚举选哪个】这两种思路，

[1,6,7,2,4,5,3]

如果我们倒着思考，假设3是子序列的最后一个数，考虑选和不选的话，这前面的数字就需要和3比较大小，所以我们不仅需要知道当前的下标，还需要知道上一个数字的下标，而如果考虑枚举选哪个，我们就可以直接枚举前面的比3小的数字，当作子序列的倒数第二个数，那么就只需要知道当前所选数字的下标就好了，所以【枚举选哪个】只需要一个参数，更加好写。

🎈启发思路：

枚举nums[i]作为LIS的末尾元素，那么需要枚举nums[j]作为LIS的倒数第二个元素，其中j<i且nums[j]<nums[i]。

🎈回溯三问：

  • 子问题：以nums[i]结尾的LIS长度
  • 当前操作：枚举nums[j]
  • 下一个子问题：以nums[j]结尾的LIS长度

dfs(i) = max{dfs(j)} + 1 j<i且nums[j]<nums[i]

把子问题求一个最大值，在加上nums[i]这一个数字

🎈code：

[1,6,7,2,4,5,3]

  • 从数字3开始递归，下一个数字是2
  • 从数字5开始递归，下一个数字也是2

这样就会出现重复子问题的现象，所以我们可以用动态规划来优化，对应到代码上就是改成记忆化搜索。

时间复杂度：$O(n)$个状态，每个状态$O(n)$的时间，所以就是$O(n^2)$

空间复杂度：由于有$O(n)$个状态，所以空间复杂度$O(n)$

把DFS函数改成数组，把递归改成循环就可以了。

f[i] = max{f[j]}+1 j<i且nums[j]<nums[i]

时间空间复杂度一样！

🎈递归分析：

递归函数

上图展现此题的状态机。

需要取利润最大值，这里有两个情况，一个是第i天未持有股票，一个是第i天持有了股票，这两个个最终也要取最大值。

当第i天未持有和持有股票的时候，第i-1天也是两种情况：

dfs(i,0) = max( dfs(i-1,0),dfs(i-1,1)+prices[i] )

dfs(i,1) = max( dfs(i-1,1),dfs(i-1,0)-prices[i] )

递归边界

  1. dfs(-1,0)=0：第0天开始的时候未持有股票，那么利润肯定为0。
  2. dfs(-1,1)=-无穷：第0天开始是不会持有股票的，利润定义为无穷小，方便后续进行max取值。

这里的i，我们填写了-1，因为i为-1作为边界的话，此时prices[]数组可以是空的，

如果i=0为边界的话，必须保证prices[]数组不能为空。

递归入口

最后一天如果持有股票的话，是卖不出去的，肯定是白白浪费钱了，所以认为

max(dfs(n-1,0),dfs(n-1),1) = dfs(n-1,0)

直接从dfs(n-1,0)开始递归就可以了！！！

🎈1:1翻译成递推：

转移方程和dfs的写法一样：

f[i][0] = max(f[i-1][0],f[i-1][1]+prices[i])

f[i][1] = max(f[i-1][1],f[i-1][0]-prices[i])

但这样会发现没有状态表示f[-1][0]和 f[-1][1]

那就在f的最前面插入一个状态

f[0][0] = 0

f[0][1] = -无穷

f[i+1][0] = max(f[i][0],f[i][1]+prices[i])

f[i+1][1] = max(f[i][1],f[i][0]-prices[i])

最后返回f[n][0]

🎈空间优化：

由于f[i+1]只用到f[i]这个状态，所以只需要一个长为二的数组滚动计算就好了，或者两个变量就可以了。

二叉树的直径

543. 二叉树的直径 - 力扣（Leetcode）

🎈思路：

从题目中可以看出二叉树的直径就是：从一个叶子出发向上，在某个节点拐弯，向下达到另一个叶子，得到了由两条链拼起来的路径，可能会出现只有一条链的现象。

既然直径会在某个节点拐弯，那么我们就可以枚举每个节点，假设直径在这里拐弯，那么就需要知道这个节点的左右子树的最长链的长度，最长链的计算方式也就是二叉树的最大深度。

🎈算法：

遍历二叉树，在计算最长链的同时，顺便把直径算出来。

在当前节点拐弯的直径长度=左子树的最长链+右子树的最长链+2。

返回给父节点的就是：

以当前节点为根的子树的最长链=max(左子树的最长链+右子树的最长链)+1

时间复杂度：O(n)【每个节点只遍历了一次】

空间复杂度：O(n)【由于在最坏的情况下，这棵二叉树是一条链，所以递归需要O(n)的栈空间】

这里没有对节点重复使用，每个节点只遍历了一次，所以不需要记忆化搜索。

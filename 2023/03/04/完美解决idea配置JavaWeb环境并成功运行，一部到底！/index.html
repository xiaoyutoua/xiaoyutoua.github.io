完美解决idea配置JavaWeb环境并成功运行，一部到底！

发表于2023-03-04|更新于2023-05-29|教程
|字数总计:1.5k|阅读时长:5分钟|阅读量:

本博主看到别人帖子总结的，自用！

前言

搞了一个晚上终于成功写出了JavaWeb文件

配置环境真的真的太难受了，这真是每次想学习一个新的东西的时候最不想经历的。

在之前学习JavaSE的时候，没有根据老师的给的idea和jdk的版本来下载，直接去官网和网上找到了最新的版本，然后大二下学期开JavaWeb课的时候的时候环境一直配不好。idea2022版我根本是配置不了这个web环境，不是缺这就缺那，那么干脆直接降低idea的版本，下面为降低版本，并进行相关web文件的创建和环境配置的过程整理，方便日后查看😪。

那么到底什么是tomcat呢？？？

tomcat是一个web服务器，我们可以将html文件部署到这个web服务器上，然后我们在访问此html页面时就可以通过访问一个类似网址的东西访问，所有你在访问此页面的时候，tomcat不能关闭。

方法一

一、降低idea版本

直接删除电脑原有的2022版本，并删除注册表中的内容。

然后去网上寻找2020年的破解版本（当然支持正版💪），直接下载即可。
我的参考：https://www.cnblogs.com/technicist/p/15229615.html

二、降低jdk版本

这里我们需要选择适合当前使用的idea版本的jdk版本。

否则我们在idea软件上运行java项目的时候会出现如下报错：

我们可以查看自己idea上对应的jdk最高的版本！！！

查看idea对应的jdk最高版本

打开IDEA 点击右上角的file 再点击Project Structure

之后点击第一个第一条，Project 查看Project language level 打开下拉条，查看你的IDEA能支持的JDK最高版本

如图所示，我的这个最高支持到14版本，所以使用JDK16是不可以的，可以下载14以下版本的JDK

在Project SDK 中选择你合适的jdk文件夹，并在Project language level中选择对应的版本号，

所以当使用jdk16时，Project language level中没有对应的16版本导致了出现这种错误

上述内容转载链接如下：

https://blog.csdn.net/WEASYD/article/details/118032654

然后直接到官网上下载相应版本的JDK，并配置环境变量JAVA_HOME等，自行去百度！
如果你之前下载了其他版本的JDK，则新下载的会自动覆盖之前的，但也要修改环境变量、注册表等等。

三、创建一个Java web项目并配置Tomcat服务器

下述内容转载链接如下：

是小崔崔崔崔崔

https://blog.csdn.net/weixin_43716048/article/details/108639475

1、创建Java Web项目

打开IDEA之后新建一个普通的Java项目

选中红框之后点next

选择位置,工程名.然后点finish

重点:找到刚才新建的Java项目,然后右键,点击 Add Framework Support...

然后勾选Web Application,点击ok

这个时候Java web项目就创建好了

2、配置Tomcat

首先直接去官网下载对应版本的Tomcat，然后配置相应的环境变量！

点击导航栏的Run->Edit Configurations...

也可以直接点击绿色小锤子🔨旁边的结构配置即可。

点击+添加，下滑找到Tomcat（注意这里不是tomEE，然后一定要选择Local📌）

按照图上的标注进行选择，其中Configure...点后要选中Tomcat的安装路径

下方有报错的话直接点击Fix，由于缺少一个jar包导致，点击Fix之后能直接自动获取。

注意📌：更改Application context为/+项目名，例如项目名如果为text，则为/text

然后点击ok即配置成功。

3、配置完成之后运行项目

点项目根文件，然后点击Run Tomcat 8.5.57

注意:在运行Java Web项目的时候，不要提前运行了tomcat🐱

注意:每次在建立一个新的Java Web项目的时候

4、报错解决方法

4.1 IntelliJ IDEA关于“cannot resolve symbol servlet”的解决

如果我们遇到了cannot resolve symbol servlet，如下图所示

出现这个问题的原因是缺少servlet-api.jar包造成的。

  1. 右键项目->open module setting

  2. libraries-> + ->Java->选择路径（tomcat安装目录下lib->tomcat-api.jar）

完成之后就没有cannot resolve symbol servlet的提示错误了

方法二

一、新建Web项目

选择IDEA主页的“Create New Project”选项，进入新建项目的界面。

在New Projec界面中，选择左侧栏的“Java”选项，然后勾选”Web Application”选项。选择完毕之后，单击”Next”按钮进入填写项目信息的界面。

在New Projec界面中，”Project name”选项用于指项目的名称，”Project localtion”选项用于指定Web项目的根目录。这里采用默认设置的目录，将chapter04作为Web项目的名称。设置完成之后，单击“Finish”按钮，进入开发界面。

二、添加Tomcat的Servlet-api.jar包

创建好Web项目后，接下来，需要在项目中添加Tomcat的Servlet-api.jar包，单击IDEA开发工具右上角的“ ”按钮，进入Project Structure界面。

在Project Structure界面中，单击【Libraries】→【+】→【Java】，弹出Select Library Files界面。

在Select Library Files界面，选择项目所在的目录后单击”OK”按钮，弹出选择项目类型界面。

在选择项目类型界面，选中Classes选项，单击“OK”按钮，会显示项目名称界面“Choose Modules”，在“Choose Modules”界面直接单击”OK”按钮，此时项目chapter04就加载到IDEA开发工具中了。

单击标记的“+”按钮，查找Tomcat 包下lib包的Servlet-api.jar，将其导入到chapter04项目中。

在单击”OK”按钮后，就可以将Servlet-api.jar添加到chapter04的项目中。

右击chapter04项目的src文件，选择【new】→【Create New Servlet】选项，进入创建Servlet类的界面。

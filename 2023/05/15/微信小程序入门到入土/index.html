1.3 小程序项目代码组成

  1. pages 用来存放所有小程序的页面
  2. utils 用来存放工具性质的模块（例如：格式化时间的自定义模块）
  3. app.js 小程序项目的入口文件
  4. app.json 小程序项目的全局配置文件
  5. app.wxss 小程序项目的全局样式文件
  6. project.config.json 项目的配置文件
  7. logs日志页面

1.3.1 pages文件夹

小程序中所有的页面都存放在pages目录中，每个页面以单独的文件夹存。

每个页面文件夹中包含四个页面，分别是：

  • .json文件：页面的配置文件，配置窗口外观等
  • .wxml文件：页面的模板文件
  • .css文件：页面的样式文件
  • .js文件：页面的脚本文件，存放页面的数据、事件处理函数等

1.3.2 json文件

json文件中的都是一个键值对。

键值对的键和值之间用：隔开，值的数据类型有多种，数组、整形、字符串等等，每个键值对之间用,隔开。

app.json

全局页面的配置文件。

pages：页面路径列表，即小程序所有页面的路径，谁放在第一个，则默认是首页，即小程序编译后第一个显示的页面，可以通过工具栏改变编译模式改变。

添加页面：在pages的数组中添加一个页面路径，小程序会自动生成页面的一个文件夹，里面包含了四个文件

删除页面：删除页面的文件夹，然后删除pages中的路径。

window：小程序的全局窗口属性，如导航栏的颜色、背景色。

tabBar：小程序底部tab栏，包括tab的列表、颜色、图标等，需要建立一个存放图标的文件夹。

每个页面单独的json

用来配置单独某个页面的窗口外观，会覆盖app.json。

1.3.3 WXML文件

一个模板文件，类似vue。

作用类似于HTML，但又要区别。

数据动态绑定、支持列表渲染、条件判断等等。

1.3.4 WXSS文件

与css一样，但尺寸单位新增了一个rpx，会根据屏幕大小的变化自动进行换算成px。

与css中%作用有点类似，但方便的许多。

其中app.wxss文件中定义的样式会作用于全局页面。

1.3.5 js文件

小程序实现的逻辑层。

app.js小程序的项目的入口文件，通过调用App()函数来启动整个小程序，不用管。

页面.js文件是每个页面的入口文件，通过Page()函数来创建并运行对应页面。

五、微信云开发

微信开放文档 (qq.com)

且后端服务选择微信云开发。

5.1 底层原理

微信云开发中的数据库底层原理使用的是腾讯云的云数据库服务。云数据库是一种基于云计算的数据库服务，提供了高可用性、高可扩展性、自动备份和恢复、自动故障转移等功能。微信云开发中的数据库服务是在腾讯云的云数据库服务上进行的封装和定制，使其更符合微信小程序的开发需求。

微信云开发中的数据库服务采用的是非关系型数据库存储模型，即NoSQL。NoSQL数据库与传统的关系型数据库不同，不使用结构化查询语言（SQL），而是使用其他数据查询语言或API来访问和管理数据。微信云开发中的数据库服务使用的是文档型数据库，即使用JSON格式存储数据，每个文档相当于一条记录。

微信云开发中的数据库服务支持实时同步和数据访问权限控制等功能。同时，它还支持云函数触发器，可以在数据库中的数据发生变化时触发云函数。这种触发器可以用于实现实时通知、自动化处理和数据分析等功能。

总的来说，微信云开发中的数据库底层原理采用了腾讯云的云数据库服务，使用文档型数据库存储模型，支持实时同步和数据访问权限控制等功能，同时还支持云函数触发器。

5.4 调用云函数进行增删查改

5.4.1 wx.cloud.callFunction

wx.cloud.callFunction 是微信小程序中调用云函数的方法，它的参数有以下几个：

  • name：云函数名称，必填项。
  • data：传递给云函数的参数，可选项。
  • config：配置选项，可选项。
  • success：成功回调函数，可选项。
  • fail：失败回调函数，可选项。
  • complete：完成回调函数，可选项。

其中，name 参数指定要调用的云函数的名称，是必填项。data 参数是传递给云函数的数据，可以是任何类型的数据，例如字符串、数字、对象等。config 参数是配置选项，可以用来设置超时时间、环境 ID 等。success、fail 和 complete 参数是回调函数，分别在调用成功、调用失败和调用完成时执行。

📌具体来说，config 参数有以下几个选项：

  • env：指定云环境 ID，如果不指定则默认使用当前小程序的云环境 ID。
  • timeout：调用云函数的超时时间，单位为毫秒，默认为 3000 毫秒。

📌​data参数具体如下：

  • action：表示要执行的操作，例如action: 'add'或者action:'delete'等等
  • data：表示要处理的数据，例如 {name: '小明', age: 18} 表示要添加的数据是一个对象，包含了名字和年龄两个属性。

5.4.2 云函数

微信小程序中的云函数是一种在云端运行的 JavaScript 代码，它可以被小程序调用，用于处理一些复杂的业务逻辑或需要访问敏感数据的操作。与传统的客户端代码不同，云函数不会被打包到小程序中，而是在云端运行，可以在多个小程序中共享使用。

通过云函数，开发者可以在小程序中使用云端数据库、云存储、云函数等云服务，并在云端完成数据的增删查改等操作。云函数可以直接调用云服务 API，也可以通过云函数 SDK 来访问云服务。

使用云函数可以将一些复杂的计算、数据处理、业务逻辑等操作从客户端移动到云端，减轻客户端的负担，提高小程序的性能和安全性。同时，云函数的运行环境是在微信的安全沙箱中，保证了代码的安全性，可以访问小程序的敏感数据而不会泄露。

使用云函数需要先在小程序管理后台开通云开发功能，并在开发工具中创建云函数。然后，开发者可以在云函数中编写 JavaScript 代码，使用云函数 SDK 访问云服务，或直接调用云服务 API。在小程序中调用云函数时，可以传递参数给云函数，云函数可以接收并处理参数，然后返回结果给小程序。

在云函数中，入口函数的参数包括 event 和 context 两个参数，它们分别代表了当前云函数被触发时的事件和上下文信息。

具体来说，event 参数是一个包含了事件信息的对象，其中包含了调用云函数时传递的参数，例如小程序调用云函数时传递的 data 参数。我们可以通过 event 参数来获取传递的数据，然后根据不同的操作进行处理。

而 context 参数是一个包含了上下文信息的对象，其中包含了当前云函数的一些元数据，例如所属环境 ID、当前时间戳、请求 ID 等。我们可以通过 context 参数来获取当前云函数的一些元信息，例如获取当前时间戳。

==sizeof()==

C++中的sizeof是一个操作符，用于计算数据类型或变量所占用的字节数。sizeof可以用于任何数据类型，包括基本数据类型、结构体、类等。

sizeof的作用有以下几个方面：

  1. 动态计算数据类型或变量所占用的内存大小，可以在编译时确定。
  2. 在分配内存时，可以使用sizeof来确定内存块的大小。
  3. 可以用来确定数组的大小，例如sizeof(arr)/sizeof(arr[0])可以计算出数组中元素的个数。
  4. 可以用来判断不同数据类型之间的差异。
  5. 在程序中使用sizeof可以提高代码的可读性和可维护性。

sizeof是一个非常有用的操作符，可以帮助程序员在编写程序时更好地控制内存的使用，提高程序的性能和可靠性。

==string& str和char* s==

string& str是一个C++中的字符串类型，它是一个对象，具有一系列的方法和属性，支持动态内存分配和自动释放等功能。它可以使用运算符重载来实现字符串的操作，如连接、比较、查找等。

而char* s是一个C语言中的字符指针，它是一个简单的指针类型，指向一个以’\0’结尾的字符数组。它不能自动分配或释放内存，需要手动管理内存空间。它的操作需要使用一些C语言中的库函数，如strcpy、strcat、strcmp等。

因此，string& str和char* s的区别在于，前者是一个对象，支持动态内存分配和自动释放，后者是一个简单的指针类型，需要手动管理内存空间。前者更加高级和安全，后者更加底层和灵活。

可以看出，使用string& str可以直接使用运算符重载来进行字符串的操作，而使用char* s需要使用库函数来实现字符串的操作。同时，使用string& str可以避免手动管理内存，更加安全和高级。

==函数参数传入对象引用与直接传入区别==

参数&v表示传递的是vector对象的引用，而不是对象本身的副本【拷贝】。使用引用作为参数可以避免函数调用时发生不必要的拷贝，提高程序的运行效率。如果使用v作为参数，则会产生一次拷贝构造，增加系统开销和时间消耗。同时，使用引用作为参数可以保证函数内部对参数的修改能够反映在函数外部。

上述printVector01函数传入的对象中用const修饰，表示了此对象不可更改，此时必须要将迭代器改为const_iterator表示只读，才能保证不报错。此时使用*it = 10000;会发生报错。

printVector02函数传入的是对象的引用，使用*it = 10000修改 了v的值，打印发现v中的值已改变，然后调用printVector04函数打印函数外部的v，发现其结果已经改变。

printVector03函数传入的是对象本身的副本，使用*it = 20000修改 了v的值，打印发现函数内部的v中的值已改变，然后调用printVector04函数打印函数外部的v，发现其结果不变。
